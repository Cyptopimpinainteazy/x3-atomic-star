feat(x3-compiler): integrate YOLO + Loop-Pack v1 optimizer into blockchain compiler pipeline

## Summary

Successfully integrated the YOLO + Loop-Pack v1 optimization pipeline into the X3 Chain
blockchain's X3 smart contract compiler. This completes Phase 4 of the optimization roadmap.

## Architecture

Created new `x3-compiler` crate that orchestrates the full compilation pipeline:

```
HIR → MIR → [OPTIMIZER (14-pass YOLO + Loop-Pack)] → Bytecode
```

The optimizer is configurable via `OptLevel` enum:
- O0: No optimization (baseline)
- O1: Basic (2 passes)
- O2: Default (14 passes: YOLO + Loop-Pack) ← RECOMMENDED
- O3: Aggressive (14 passes, up to 20 iterations)

## Implementation Details

### New Module: crates/x3-compiler/
- `Compiler::compile_mir(mir, options) -> BytecodeModule`
- `CompilationOptions` with OptLevel configuration
- Clean separation: Compiler sits above both Backend and Optimizer
- No circular dependencies

### Public API
```rust
use x3_compiler::{Compiler, CompilationOptions};

let bytecode = Compiler::compile_mir(
    &mir,
    CompilationOptions::opt2().with_verbose(true)
)?;
```
